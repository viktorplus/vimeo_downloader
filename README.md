# Загрузчик видеоуроков с Vimeo

Скрипт автоматически скачивает видеоуроки с Vimeo, включая приватные ссылки с токеном (вида `vimeo.com/ID/токен?fl=pl&fe=cm`). Заменяет ручное использование сторонних сервисов типа catch.tube.

Включает веб-интерфейс на `http://127.0.0.1:5050` — никакого Apache, Nginx или хостинга не нужно. Встроенный сервер Flask запускается прямо на вашем компьютере и работает только локально.

---

## Быстрый старт (Windows)

### Шаг 1 — Установить Python

Скачать с [python.org](https://www.python.org/downloads/) версию **3.10 или новее**.

При установке обязательно поставить галочку **"Add Python to PATH"**.

Проверить установку:

```powershell
python --version
```

### Шаг 2 — Установить ffmpeg

ffmpeg нужен для объединения видео и аудио в один MP4-файл.

```powershell
winget install ffmpeg
```

Или скачать вручную с [ffmpeg.org](https://ffmpeg.org/download.html) и добавить папку `bin` в системный PATH.

Проверить:

```powershell
ffmpeg -version
```

### Шаг 3 — Скачать проект

```bash
git clone https://github.com/viktorplus/vimeo_downloader.git
cd vimeo_downloader
```

Или скачать ZIP с GitHub: `Code → Download ZIP`, распаковать.

### Шаг 4 — Создать виртуальное окружение

Виртуальное окружение изолирует зависимости проекта от системного Python — это стандартная практика.

```bash
python -m venv .venv
```

### Шаг 5 — Установить зависимости

```bash
.venv\Scripts\pip install -r requirements.txt
```

Это установит:

- **flask** — встроенный веб-сервер для интерфейса на 127.0.0.1:5050
- **yt-dlp** — загрузчик видео с Vimeo
- **selenium** — автоматизация браузера для синхронизации уроков из LMS
- **curl-cffi** — дополнительный транспорт для обхода защиты Vimeo

### Шаг 6 — Запустить веб-интерфейс

```bat
start_lessons_menu.bat
```

Или вручную:

```bash
.venv\Scripts\python lessons_menu.py --host 127.0.0.1 --port 5050
```

Откройте браузер и перейдите по адресу:

```text
http://127.0.0.1:5050
```

> **Никакой дополнительный сервер устанавливать не нужно.** Flask — это встроенный веб-сервер, он запускается вместе со скриптом. Сайт работает только на вашем компьютере, снаружи он недоступен.

## Веб-интерфейс — что умеет

- Список всех уроков из `lessons_list.txt`
- Фильтр по предмету и поиск по названию
- Выбор качества для каждого урока (360p — 2160p)
- Кнопка `Скачать` у каждой записи
- Кнопка `Обновить список из LMS` — подтягивает новые уроки без дублей

## Автоматический режим (sync + download)

Если в файле `.env` указаны логин/пароль и настройки качества — скрипт сам залогинится в LMS, обновит список уроков и скачает всё, что появилось нового. Никаких кликов в браузере и ввода Enter.

**Шаг 1 — создать `.env`:**

```bash
copy .env.example .env
```

Открой `.env` и заполни:

```text
LMS_LOGIN=твой_email@example.com
LMS_PASSWORD=твой_пароль
QUALITY=720
FAST=1
```

- `QUALITY` — 360, 480, 720, 1080, 1440, 2160
- `FAST=1` — быстрое скачивание одним потоком (немного ниже качество, но без ffmpeg)
- `FAST=0` — лучшее качество с раздельным скачиванием видео+аудио и склейкой через ffmpeg

> **Безопасность:** `.env` не попадает в git (исключён в `.gitignore`). Пароль остаётся только на твоём компьютере.

**Шаг 2 — запустить:**

```bat
auto_sync_and_download.bat
```

Скрипт:

1. Откроет браузер (Edge), залогинится сам по `.env`
2. Откроет страницу записей LMS, дождётся загрузки таблицы
3. Распарсит список, добавит в `lessons_list.txt` новые уроки
4. Скачает все уроки, которых нет в `.downloaded` архиве, с настройками из `.env`
5. Выдаст финальный отчёт: сколько скачано, пропущено, упало с ошибкой

**Когда применять:**

- Раз в неделю/месяц для подтягивания новых записей
- После каникул / долгого перерыва — за один запуск догонит всё

## Обновить список уроков из LMS

Нужен аккаунт на [lms.itcareerhub.de](https://lms.itcareerhub.de).

> Если в `.env` указаны `LMS_LOGIN`/`LMS_PASSWORD` — `update_lessons_list.bat` тоже использует автологин и не просит нажать Enter.

**Вариант 1 — прямо в меню:**

1. Открыть `http://127.0.0.1:5050`
2. Нажать кнопку `Обновить список из LMS`
3. Откроется браузер — войдите в кабинет, если потребуется
4. Дождаться сообщения о завершении — страница обновится автоматически

**Вариант 2 — отдельным скриптом:**

```bat
update_lessons_list.bat
```

1. Откроется браузер на странице записей LMS
2. Войдите в кабинет (если нужно)
3. Нажмите `Enter` в консоли
4. Скрипт вытащит уроки из таблицы и допишет в `lessons_list.txt` только новые записи

После завершения перезапустите `start_lessons_menu.bat`, чтобы увидеть новые уроки.

> **Примечание:** Для этой функции нужен браузер **Microsoft Edge** (уже встроен в Windows 10/11). Selenium управляет им автоматически — ничего дополнительно устанавливать не нужно.

**Вариант 3 — парсинг локально сохранённой страницы (fallback):**

Если LMS временно недоступен или Selenium не работает, можно взять уроки из HTML-страницы, сохранённой вручную из любого браузера.

Как сохранить страницу:

1. Открыть `https://lms.itcareerhub.de/local/airtable_schedule/records.php` в обычном браузере
2. Войти в кабинет
3. **Дождаться полной загрузки таблицы** (важно — уроки подгружаются через JavaScript)
4. Нажать `Ctrl+S` → в диалоге выбрать формат **"Веб-страница, только HTML"** (не "полностью")
5. Сохранить в любое место, например `d:\dev\vimeo_downloader\saved_records.html`

Запуск парсера:

```bash
.venv\Scripts\python update_lessons_list.py --html-file "saved_records.html"
```

Скрипт достанет из файла все Vimeo-ссылки, названия, предметы и даты — потом допишет в `lessons_list.txt` только новые записи. Никакого браузера, никакого логина, никаких cookies.

## Скачать одно видео (без интерфейса)

```bash
.venv\Scripts\python download_lessons.py "https://vimeo.com/1177373233/a17469db60?fl=pl&fe=cm"
```

### Указать качество

```bash
.venv\Scripts\python download_lessons.py "https://vimeo.com/..." --quality 720
```

Доступные значения: `360`, `480`, `720`, `1080` (по умолчанию), `1440`, `2160`

### Указать папку сохранения

```bash
.venv\Scripts\python download_lessons.py "https://vimeo.com/..." --output "D:/Уроки/Python"
```

### Скачать список уроков из файла

Создать текстовый файл `urls.txt` — по одной ссылке на строку. Строки с `#` — комментарии, игнорируются.

```text
# Python курс
https://vimeo.com/111111111/aabbccdd?fl=pl&fe=cm
https://vimeo.com/222222222/eeff0011?fl=pl&fe=cm

# QA курс
https://vimeo.com/333333333/112233aa?fl=pl&fe=cm
```

```bash
.venv\Scripts\python download_lessons.py --file urls.txt --quality 1080 --output "D:/Уроки"
```

### Посмотреть доступные форматы без скачивания

```bash
.venv\Scripts\python download_lessons.py "https://vimeo.com/..." --list-formats
```

## Все параметры

| Параметр             | Короткий | Описание                                             |
|----------------------|----------|------------------------------------------------------|
| `url`                |          | URL видео (позиционный аргумент)                     |
| `--file ФАЙЛ`        | `-f`     | Текстовый файл со списком URL                        |
| `--quality КАЧЕСТВО` | `-q`     | Максимальное разрешение (360/480/720/1080/1440/2160) |
| `--output ПАПКА`     | `-o`     | Папка для сохранения файлов                          |
| `--list-formats`     | `-l`     | Показать форматы без скачивания                      |

## Часто задаваемые вопросы

**Нужен ли хостинг или отдельный сервер для интерфейса?**  
Нет. Flask — это встроенный веб-сервер, он запускается прямо на вашем компьютере. `http://127.0.0.1:5050` — это адрес вашего же компьютера (localhost), снаружи он недоступен.

**Почему скрипт, а не catch.tube?**  
Ручной процесс: открыть сайт → вставить ссылку → выбрать качество → скачать.  
Со скриптом: открыть интерфейс → выбрать урок → нажать кнопку.

**Работают ли приватные ссылки с токеном?**  
Да. Ссылки вида `vimeo.com/ID/токен?fl=pl&fe=cm` работают напрямую.

**Что делать если ошибка "ffmpeg not found"?**  
Установить ffmpeg (Шаг 2). После установки закрыть и снова открыть консоль — Windows обновляет PATH только при новом запуске.

**Ошибка "python не является внутренней командой"?**  
Python не добавлен в PATH. Переустановить Python с галочкой "Add Python to PATH", или использовать `py` вместо `python`.

**Можно ли скачать в формате, отличном от MP4?**  
Скрипт принудительно объединяет в MP4. Для изменения — отредактировать параметр `merge_output_format` в функции `download()` в файле `download_lessons.py`.

**Что такое виртуальное окружение (.venv)?**  
Изолированная папка с Python и зависимостями именно для этого проекта. Не влияет на остальные программы. Если удалить папку `.venv` — просто пересоздать по Шагу 4–5.
